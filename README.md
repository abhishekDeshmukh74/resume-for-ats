# Resume for ATS — AI Resume Tailor

Upload your resume (PDF or LaTeX), provide a job description (paste or URL), and get an ATS-optimised resume generated by a **LangGraph multi-agent pipeline** with parallel section rewriters and a conditional refinement loop. The output preserves your original PDF formatting — only the text content is updated.

## Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | React 19 + Vite 8 (TypeScript) + Tailwind CSS 4 + Zustand 5 |
| Backend | Python + FastAPI |
| AI | LangGraph multi-agent pipeline · 7 LLM providers via **litellm** |
| ATS Scoring | Algorithmic via `rapidfuzz` (exact, synonym, fuzzy) + section weighting |
| PDF | PyMuPDF (in-place text rewriting) · LaTeX compilation via xelatex/pdflatex |
| Database | MongoDB (pipeline run tracking — optional) |
| Routing | react-router-dom 7 |

## Setup

### Prerequisites

- Python 3.11+
- Node.js 20+
- An API key for at least one LLM provider (e.g. [Groq](https://console.groq.com/), [Google Gemini](https://aistudio.google.com/))
- A LaTeX distribution (optional — for `.tex` resume compilation; [MiKTeX](https://miktex.org) or TeX Live)
- MongoDB instance (optional — for pipeline run tracking)

### 1. Clone & configure

```bash
git clone https://github.com/abhishekDeshmukh74/resume-for-ats.git
cd resume-for-ats
cp .env.example .env
# Edit .env — set at least one API key and LLM_PROVIDER, optionally MONGODB_URL
```

### 2. Backend

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate

pip install -r backend/requirements.txt
```

### 3. Frontend

```bash
cd frontend
npm install
```

## How It Works

1. **Upload Resume** — Drag-drop or select a `.pdf` or `.tex` file. The backend extracts text and preserves the original file.
2. **Job Description** — Paste the JD text, or provide a URL and the backend scrapes it.
3. **Generate** — The AI pipeline analyses, rewrites (3 sections in parallel), validates, scores, and optionally refines your resume against the JD.
4. **Review Diffs** — Preview proposed old→new text changes before applying them.
5. **Download** — Preview the final tailored PDF with before/after ATS score comparison, download it, and optionally generate a cover letter + LinkedIn message.

### AI Pipeline

```
extract_keywords → analyse_resume → score_before
  → ┌ rewrite_skills  ┐
    │ rewrite_summary  │ (parallel)
    └ rewrite_experience┘
  → qa_deduplicate → score_extract
  → [if score < 90: refine_rewrite → refine_qa]
  → compile_pdf
```

| # | Agent | Purpose |
|---|-------|---------|
| 1 | Keyword Extractor | Extract 30–60 categorised keywords from the JD (required vs preferred) |
| 2 | Resume Analyser | Map resume sections, identify keyword gaps with placement instructions |
| 3 | Pre-Rewrite Scorer | Baseline ATS score using algorithmic matching (rapidfuzz) + LLM |
| 4a | Skills Rewriter | Add missing keywords to skills lines (comma-separated format) |
| 4b | Summary Rewriter | Rewrite summary with broad JD keywords (natural prose) |
| 4c | Experience Rewriter | Rewrite experience bullets with specific keywords (preserve metrics) |
| 5 | QA Agent | Validate replacements, deduplicate keywords, clean AI phrases |
| 6 | Final Scorer | Algorithmic ATS score + structured data extraction |
| 6b | Refinement Writer | (conditional) Inject still-missing keywords if score < 90 |
| 7 | PDF Compiler | Apply replacements to original file and produce final PDF |

See [backend/services/agents/AGENTS.md](backend/services/agents/AGENTS.md) for full details.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/parse-resume` | Upload PDF/LaTeX → extracted text + base64 file |
| `POST` | `/api/scrape-jd` | `{url}` → scraped JD text |
| `POST` | `/api/generate-resume` | Full pipeline → `ResumeData` + rewritten PDF (base64) |
| `POST` | `/api/generate-resume-stream` | SSE streaming full pipeline |
| `POST` | `/api/preview` | Pipeline without PDF compilation → proposed replacements |
| `POST` | `/api/preview-stream` | SSE streaming preview pipeline |
| `POST` | `/api/confirm` | Apply approved replacements → compiled PDF |
| `POST` | `/api/generate-cover-letter` | Cover letter + suggested title + LinkedIn message |
| `GET` | `/api/pipeline-runs` | List recent pipeline runs (`?limit=` `?skip=`) |
| `GET` | `/api/pipeline-runs/{id}` | Full detail of a pipeline run |
| `GET` | `/api/pipeline-runs/{id}/pdf` | Download compiled PDF |
| `GET` | `/api/pipeline-runs/status` | Check MongoDB connectivity |
| `GET` | `/api/health` | Health check |

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `LLM_PROVIDER` | No | `groq` (default), `gemini`, `openai`, `anthropic`, `ollama`, `deepseek`, `openrouter` |
| `GROQ_API_KEY` | Yes* | API key from [console.groq.com](https://console.groq.com/) |
| `GROQ_API_KEYS` | No | Comma-separated Groq keys for round-robin fallback |
| `GEMINI_API_KEY` | Yes* | API key from [aistudio.google.com](https://aistudio.google.com/) |
| `OPENAI_API_KEY` | Yes* | OpenAI API key |
| `ANTHROPIC_API_KEY` | Yes* | Anthropic API key |
| `DEEPSEEK_API_KEY` | Yes* | DeepSeek API key |
| `OPENROUTER_API_KEY` | Yes* | OpenRouter API key |
| `OLLAMA_BASE_URL` | No | Ollama server URL (default: `http://localhost:11434`) |
| `GROQ_MODEL` | No | Groq model name (default: `llama-3.3-70b-versatile`) |
| `GEMINI_MODEL` | No | Gemini model name (default: `gemini-2.0-flash`) |
| `MONGODB_URL` | No | MongoDB connection string for pipeline run tracking |
| `ALLOWED_ORIGINS` | No | Comma-separated CORS origins (default: `http://localhost:5173`) |
| `LATEX_COMPILER_PATH` | No | Override path to xelatex/pdflatex binary |

\* At least one API key is required, matching the selected `LLM_PROVIDER`.
