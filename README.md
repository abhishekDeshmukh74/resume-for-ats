# Pass ATS — AI Resume Tailor

Upload your resume (PDF or LaTeX), provide a job description (paste or URL), and get an ATS-optimised resume generated by a **LangGraph multi-agent pipeline** powered by Groq or Google Gemini. The output preserves your original PDF formatting — only the text content is updated.

## Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | React 19 + Vite 8 (TypeScript) + Tailwind CSS 4 |
| Backend | Python + FastAPI |
| AI | LangGraph 14-node multi-agent pipeline · Groq (`llama-3.3-70b-versatile`) or Google Gemini (`gemini-2.0-flash`) |
| PDF | PyMuPDF (in-place text rewriting) · LaTeX compilation via xelatex/pdflatex |
| Database | MongoDB (pipeline run tracking — optional) |
| Routing | react-router-dom |

## Setup

### Prerequisites

- Python 3.11+
- Node.js 20+
- A free [Groq API key](https://console.groq.com/) or [Google Gemini API key](https://aistudio.google.com/)
- A LaTeX distribution (optional — for `.tex` resume compilation; [MiKTeX](https://miktex.org) or TeX Live)
- MongoDB instance (optional — for pipeline run tracking)

### 1. Clone & configure

```bash
git clone https://github.com/abhishekDeshmukh74/pass-ats.git
cd pass-ats
cp .env.example .env
# Edit .env — set GROQ_API_KEY or GEMINI_API_KEY, ALLOWED_ORIGINS, and optionally MONGODB_URL
```

### 2. Backend

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate

pip install -r backend/requirements.txt
```

### 3. Frontend

```bash
cd frontend
npm install
```

## How It Works

1. **Upload Resume** — Drag-drop or select a `.pdf` or `.tex` file. The backend extracts text and preserves the original file.
2. **Job Description** — Paste the JD text, or provide a URL and the backend scrapes it.
3. **Generate** — A 14-node LangGraph pipeline parses, analyses, optimises, validates, scores your resume against the JD, and compiles the final PDF. For PDF uploads the rewritten text is substituted in-place; for LaTeX uploads the source is patched and compiled.
4. **Preview & Download** — Preview the tailored PDF inline with a before/after ATS score comparison, then download it.

### AI Pipeline (14 Nodes)

| # | Agent | File | Purpose |
|---|-------|------|---------|
| 1 | Intake Parser | `intake_parser.py` | Parse raw resume text into structured JSON |
| 2 | JD Analyzer | `jd_analyzer.py` | Extract hiring signals, required/preferred skills, ATS keywords from the JD |
| 3 | Gap Analyzer | `gap_analyzer.py` | Compare resume vs JD — identify missing keywords and weak areas |
| 4 | Baseline Scorer | `scorer.py` | Compute baseline ATS score before any changes |
| 5 | Summary Optimizer | `summary_optimizer.py` | Rewrite professional summary to align with JD |
| 6 | Skills Optimizer | `skills_optimizer.py` | Reorder and supplement skills based on JD priority |
| 7 | Bullet Rewriter | `bullet_rewriter.py` | Rewrite experience bullets (Action + Tech + Scope + Result) |
| 8 | Dedup Optimizer | `dedup_optimizer.py` | Remove duplicate keywords across bullets |
| 9 | Truth Guard | `truth_guard.py` | Verify all claims against the original resume |
| 10 | Critic | `critic.py` | Quality gate — pass or route back for targeted revision (max 2 loops) |
| 11 | Final Scorer | `scorer.py` | Score the rewritten resume, extract structured data |
| 12 | Formatter | `formatter.py` | Export resume in text/markdown formats with old→new replacements |
| 13 | PDF Compiler | `pdf_compiler.py` | Apply replacements to original file and produce final PDF |

See [backend/services/agents/AGENTS.md](backend/services/agents/AGENTS.md) for full details including the revision loop logic and scoring formula.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/parse-resume` | Upload PDF/LaTeX → extracted text + base64 file |
| `POST` | `/api/scrape-jd` | `{url}` → scraped JD text |
| `POST` | `/api/generate-resume` | Resume + JD → AI-tailored `ResumeData` + rewritten PDF (base64) |
| `GET` | `/api/pipeline-runs` | List recent pipeline runs (with `?limit=` and `?skip=`) |
| `GET` | `/api/pipeline-runs/{id}` | Full detail of a single pipeline run |
| `GET` | `/api/pipeline-runs/{id}/pdf` | Download compiled PDF for a pipeline run |
| `GET` | `/api/health` | Health check |

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GROQ_API_KEY` | Yes* | API key from [console.groq.com](https://console.groq.com/) |
| `GEMINI_API_KEY` | Yes* | API key from [aistudio.google.com](https://aistudio.google.com/) |
| `LLM_PROVIDER` | No | `"groq"` (default) or `"gemini"` |
| `GROQ_MODEL` | No | Groq model name (default: `llama-3.3-70b-versatile`) |
| `GEMINI_MODEL` | No | Gemini model name (default: `gemini-2.0-flash`) |
| `ALLOWED_ORIGINS` | Yes | Comma-separated list of allowed CORS origins (e.g. `http://localhost:5173`) |
| `MONGODB_URL` | No | MongoDB connection string for pipeline run tracking |

\* At least one of `GROQ_API_KEY` or `GEMINI_API_KEY` is required, depending on `LLM_PROVIDER`.
